Chcesz znów poeksperymentować. Robisz więc zmiany na eksperymentalnym branczu. Tym razem chcesz żeby po udanym eksperymencie zmiany znalazły się chronologicznie po wszystkich zmianach na branczu głównym.

Użyj repozytorium z poprzedniego ćwiczenia.

$ git checkout -b experiment-2
odpowiedź:
Switched to a new branch 'experiment-2'

Utwórz nowy lokalny brancz o nazwie experiment-2. Komenda checkout z przełącznikiem -b utworzy brancz i od razu przeniesie cię na niego. 

$ git status
odpowiedź:
On branch experiment-2
Changes not staged for commit:
    (use "git add <file>..." to update what will be committed)
    (use "git checkout -- <file>..." to discard changes in working directory)

modified:   about.txt

Untracked files:
    (use "git add <file>..." to include in what will be committed)

Nowy dokument tekstowy.txt

no changes added to commit (use "git add" and/or "git commit -a")

Wprowadź dowolne zmiany w plikach repozytorium. Zobacz jaki jest teraz stan repozytorium za pomocą polecenia status. 

$ git add .

$ git status
odpowiedź:
On branch experiment-2
Changes to be committed:
  (use "git reset HEAD <file>..." to unstage)

new file:   Nowy dokument tekstowy.txt
modified:   about.txt

Dodaj zmiany do indeksu poleceniem add . i sprawdź ponownie stan. 

$ git commit -m "Mój komit mesydż"
odpowiedź:
[experiment-2 deed0580] Mój komit mesydż
 2 files changed, 3 insertions(+), 1 deletion(-)
 create mode 100644 Nowy dokument tekstowy.txt

Zakomituj zmiany z dowolnym mesydżem. 

$ git checkout master

W tak zwanym międzyczasie, potrzebujesz zrobić drobne poprawki na głównym branczu. Przejdź na brancz główny poleceniem checkout. 

$ git do it --yourself ;)

Zrób dowolną zmianę, dodaj do indeksu, zakomituj i pusznij, te polecenia już znasz. 

$ git checkout experiment-2

$ git rebase master
odpowiedź:
First, rewinding head to replay your work on top of it...
Applying: Komit na eksperymencie

Twój eksperyment okazał się sukcesem, chcesz więc go dołączyć do głównego brancza i pracować dalej. Chcesz aby zmiany eksperymentalne znalazły się PO wszystkich zmianach na branczu głównym.
Żeby to osiągnąć trzeba brancz eksperymentalny zribejzować względem brancza master. Użyj komendy rebase będąc na branczu eksperymentalnym. 

$ git checkout master 

$ git merge experiment-2
odpowiedź:
Updating a77dc6a..7068555
Fast-forward
 1_git_solo/cwiczenie_1.3.html   | 113 ++++++++++++++++++++++++++++++++++++++++
 1_git_solo/fork.after.merge.png | Bin 0 -> 46155 bytes
 2 files changed, 113 insertions(+)
 create mode 100644 1_git_solo/cwiczenie_1.3.html
 create mode 100644 1_git_solo/fork.after.merge.png

Teraz można te zmiany domerdżować do brancza głównego. Przejdź do brancza master i zmerdżuj brancz experiment-2.
Zauważ, że Git nie poprosił teraz o mesydż.

Na grafie w dowolnym GUI zobacz jak rysuje się graf po takim merdżu. Jaka jest różnica względem poprzedniego ćwiczenia?
