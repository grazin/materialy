Rozpocząłeś/rozpoczęłaś pracę nad samodzielnym projektem. Jesteś w punkcie, w którym chcesz poeksperymentować. Chcesz więc zabezpieczyć aktualny stan projektu, żeby ewentualnie do niego wrócić i zacząć inny eksperyment.

Użyj repozytorium z poprzedniego ćwiczenia.

$ git add .

$ git commit -m "Drugi commit"

$ git push origin master

Stwórz jeszcze jeden komit, tak, żeby mieć co najmniej 2. Dodaj nowy plik z dowolną treścią, następnie git add ., git commit i git push, tym razem już bez przełącznika -u.

$ git log --oneline
odpowiedź:
3f3e187 (HEAD -> master, origin/master) Drugi commit
13bb5ed Iniszial komit mesydż

Historię możesz podejrzeć poleceniem git log z przełęcznikiem --oneline jeśli chcesz tylko skróconą wersję.

$ git checkout -b experiment-1
odpowiedź:
Switched to a new branch 'experiment-1'

Utwórz nowy lokalny brancz o nazwie experiment-1. Komenda checkout z przełącznikiem -b utworzy brancz i od razu przeniesie cię na niego. 

$ git status
odpowiedź:
On branch experiment-1
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git checkout -- <file>..." to discard changes in working directory)

modified:   about.txt

Untracked files:
  (use "git add <file>..." to include in what will be committed)

Nowy dokument tekstowy.txt

no changes added to commit (use "git add" and/or "git commit -a")

Utwórz jeden nowy plik i wprowadź zmiany w już istniejącym. Zobacz jaki jest teraz stan repozytorium za pomocą polecenia status. 

$ git add .

$ git status
odpowiedź:
On branch experiment-1
Changes to be committed:
  (use "git reset HEAD <file>..." to unstage)

new file:   Nowy dokument tekstowy.txt
modified:   about.txt

Dodaj zmiany do indeksu poleceniem add . i sprawdź ponownie stan. 

$ git commit -m "Mój komit mesydż"
odpowiedź:
[experiment-1 deed0580] Mój komit mesydż
 2 files changed, 3 insertions(+), 1 deletion(-)
 create mode 100644 Nowy dokument tekstowy.txt

Zakomituj zmiany z dowolnym mesydżem. 

$ git checkout master

W tak zwanym międzyczasie, potrzebujesz zrobić drobne poprawki na głównym branczu. Przejdź na brancz główny poleceniem checkout. 

$ git do it --yourself ;)

Zrób dowolną zmianę, dodaj do indeksu, zakomituj i pusznij, te polecenia już znasz. 

$ git merge experiment-1

Twój eksperyment okazał się sukcesem, chcesz więc go dołączyć do głównego brancza i pracować dalej. Użyj komendy merge z głównego brancza.

Git zapyta o mesydż dodatkowego komita, który zostanie utworzony w skutek takiego merdża. Można ten krok pominąć, zostawiając domyślny mesydż. Po prostu wyjdź z tego trybu skrótem Shift+zz lub :wq.

Na grafie w dowolnym GUI zobacz jak rysuje się graf po takim merdżu.
