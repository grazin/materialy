Rozpocząłeś/rozpoczęłaś pracę nad samodzielnym projektem. Jesteś w punkcie, w którym chcesz poeksperymentować. Chcesz więc zabezpieczyć aktualny stan projektu, żeby ewentualnie do niego wrócić i zacząć inny eksperyment.

Użyj repozytorium z poprzedniego ćwiczenia. Stwórz jeszcze jeden komit, tak, żeby mieć co najmniej 2.

$ git status
On branch master
Your branch is up to date with 'origin/master'.

nothing to commit, working tree clean

Upewnij się, że jesteś na czysto.

$ git checkout -b experiment-1
Switched to a new branch 'experiment-1'

Utwórz nowy lokalny brancz o nazwie experiment-1. Komenda checkout z przełącznikiem -b utworzy brancz i od razu przeniesie cię na niego. 

$ git status

On branch experiment-1
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git checkout -- <file>..." to discard changes in working directory)

modified:   about.txt

Untracked files:
  (use "git add <file>..." to include in what will be committed)

Nowy dokument tekstowy.txt

no changes added to commit (use "git add" and/or "git commit -a")

Utwórz jeden nowy plik i wprowadź zmiany w już istniejącym. Zobacz jaki jest teraz stan repozytorium za pomocą polecenia status. 

$ git add .
$ git status
On branch experiment-1
Changes to be committed:
  (use "git reset HEAD <file>..." to unstage)

new file:   Nowy dokument tekstowy.txt
modified:   about.txt

Dodaj zmiany do indeksu poleceniem add . i sprawdź ponownie stan. 

$ git commit -m "Mój komit mesydż"
[experiment-1 deed0580] Mój komit mesydż
 2 files changed, 3 insertions(+), 1 deletion(-)
 create mode 100644 Nowy dokument tekstowy.txt

Zakomituj zmiany z dowolnym mesydżem. 

$ git checkout master
$ git merge experiment-1 --ff

Twój eksperyment okazał się sukcesem, chcesz więc go dołączyć do głównego brancza i pracować dalej. Użyj przełącznika --ff, który oznacza fast forward.

obrazek z gui

Zobacz na grafie w dowolnym GUI skutek tego merdża. 

$ git checkout master
$ git merge experiment-1 --no-ff

Powtórz to ćwiczenie jeszcze raz, utwórz nowy bracz na nowy eksperyment, na przykład experiment-2, ale tym razem przy merdżu użyj przełącznika --no-ff.

Git zapyta o mesydż dodatkowego komita, który zostanie utworzony w skutek takiego merdża. Można ten krok pominąć, zostawiając domyślny mesydż. Po prostu wyjdź z tego trybu skrótem Shift+zz lub :wq.

Na grafie w dowolnym GUI porównaj skutki obu merdży.
