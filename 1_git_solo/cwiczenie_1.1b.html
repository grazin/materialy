Rozpocząłeś/rozpoczęłaś już pracę nad dużym samodzielnym projektem. Chcesz, żeby twoja praca była bezpieczna i chcesz eksplorować różne podejścia do tematu.

Zacznij od utworzenia folderu z projektem z paroma plikami/folderami. Następnie otwórz konsolę git bash w folerze tego projektu 

$ git init
odpowiedź:
Initialized empty Git repository in //Documents/Politechnika - Git/repo-testy/moj projekt/.git/

Zainicjuj repozytorium poleceniem git init.

$ git status
odpowiedź:
On branch master

No commits yet

Untracked files:
    (use "git add <file>..." to include in what will be committed)

readme.txt
src/

nothing added to commit but untracked files present (use "git add" to track)

Teraz sprawdź status swojego nowego repozytorium poleceniem git status.

$ git add .
            
$ git status
odpowiedź:
On branch master
Changes to be committed:
    (use "git reset HEAD <file>..." to unstage)

new file:   readme.txt
new file:   src/siala baba mak.txt

Dodaj zmiany do indeksu poleceniem add . i sprawdź ponownie status. 

$ git commit -m "Start projektu"
odpowiedź:
[master (root-commit) b3a6c5b] Start projektu
 2 files changed, 0 insertions(+), 0 deletions(-)
 create mode 100644 readme.txt
 create mode 100644 src/siala baba mak.txt

$ git status
odpowiedź:
On branch master
nothing to commit, working tree clean

Zakomituj teraz zmiany z dowolnym mesydżem. Sprawdź ponownie status.

Póki co, twoje repozytorium istnieje tylko lokalnie. Stwórz nowe publiczne repozytorium: github.com/new, żeby spiąć je ze swoim lokalnym.

Przejdź na nowo utworzone repozytorium i skopiuj url HTTPS.

$ git remote add origin https://github.com/grazin/repo2.git

Wróć do git basha i zepnij swoje repozytorium z originem z GitHuba.

$ git push -u origin master
odpowiedź:
Enumerating objects: 4, done.
Counting objects: 100% (4/4), done.
Delta compression using up to 4 threads.
Compressing objects: 100% (2/2), done.
Writing objects: 100% (4/4), 324 bytes | 14.00 KiB/s, done.
Total 4 (delta 0), reused 0 (delta 0)
remote:
remote: Create a pull request for 'master' on GitHub by visiting:
remote:      https://github.com/grazin/repo2/pull/new/master
remote:
To github.com:grazin/repo2.git
    * [new branch]      master -> master
Branch 'master' set up to track remote branch 'master' from 'origin'.

$ git status
odpowiedź:
On branch master
Your branch is up to date with 'origin/master'.

nothing to commit, working tree clean

Teraz pusznij zmiany do origina (czyli do GitHuba). Ponieważ to pierwszy komit na tym branczu, użyj przełącznika -u. Następnie znów odczytaj status. 

$ git log
odpowiedź:
commit 13bb5ed835a5832d752d9f00e74a4732f5a8382c (HEAD -> master, origin/master)
Author: IZABELA SZCZEPANIK <izabela.szczepanik@ing.com>
Date:   Tue Nov 27 14:28:32 2018 +0100

    Start projektu

Na koniec podejrzyj historię. Użyj polecenia git log, zobacz także log w dowolnym GUI.
