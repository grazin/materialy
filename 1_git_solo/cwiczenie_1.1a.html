Chcesz rozpocząć pracę nad dużym samodzielnym projektem. Chcesz, żeby twoja praca była bezpieczna i chcesz eksplorować różne podejścia do tematu.

Jeśli jeszcze nie masz, stwórz konto na GitHubie: github.com/join

Stwórz nowe publiczne repozytorium: github.com/new

Przejdź na nowo utworzone repozytorium i skopiuj url HTTPS. Można też SSH, ale wtedy trzeba by skonfigurować klucze ssh, żeby móc się połączyć.

Otwórz konsolę git bash w wybranej lokalizacji. Jeśli nie masz menu kontekstowego, otwórz konsolę i przejdź do wybranej lokalizacji za pomocą wiersza poleceń konsoli cd.

$ git clone https://github.com/grazin/materialy.git
odpowiedź:
Cloning into 'materialy'...
Warning: Permanently added the RSA host key for IP address '140.82.118.3' to the list of known hosts.
warning: You appear to have cloned an empty repository.

Sklonuj repozytorium. Git powinien zapytać o usera i hasło do GitHuba.

$ cd materialy

$ git status
odpowiedź:
On branch master

No commits yet

nothing to commit (create/copy files and use "git add" to track)

Przejdź do sklonowanego repozytorium i odczytaj status repozytorium poleceniem git status. 

$ git status
odpowiedź:
On branch master

No commits yet

Untracked files:
    (use "git add <file>..." to include in what will be committed)

readme.txt

nothing added to commit but untracked files present (use "git add" to track)

Utwórz jeden nowy plik z dowolną treścią. Na przykład readme.txt. Zobacz jaki jest teraz status repozytorium. 

$ git add .

$ git status
odpowiedź:
On branch master
Changes to be committed:
    (use "git reset HEAD <file>..." to unstage)

new file:   readme.txt

Dodaj zmiany do indeksu poleceniem add . i sprawdź ponownie status. 

$ git commit -m "Iniszial komit mesydż"
odpowiedź:
[master (root-commit) 13bb5ed] Iniszial komit mesydż
1 file changed, 1 insertion(+)
create mode 100644 readme.txt

$ git status
odpowiedź:
On branch master
Your branch is based on 'origin/master', but the upstream is gone.
  (use "git branch --unset-upstream" to fixup)

nothing to commit, working tree clean

Zakomituj teraz zmiany z dowolnym mesydżem. Sprawdź ponownie status. 

$ git push -u origin master
odpowiedź:
Warning: Permanently added the RSA host key for IP address '140.82.118.4' to the list of known hosts.
Enumerating objects: 3, done.
Counting objects: 100% (3/3), done.
Writing objects: 100% (3/3), 263 bytes | 11.00 KiB/s, done.
Total 3 (delta 0), reused 0 (delta 0)
remote:
remote: Create a pull request for 'master' on GitHub by visiting:
remote:      https://github.com/grazin/materialy/pull/new/master
remote:
To github.com:grazin/materialy.git
    * [new branch]      master -> master
Branch 'master' set up to track remote branch 'master' from 'origin'.

$ git status
odpowiedź:
On branch master
Your branch is up to date with 'origin/master'.

nothing to commit, working tree clean

Teraz pusznij zmiany do origina (czyli do GitHuba). Ponieważ to pierwszy komit na tym branczu, użyj przełącznika -u. Następnie znów odczytaj status. 

$ git log
odpowiedź:
commit 13bb5ed835a5832d752d9f00e74a4732f5a8382c (HEAD -> master, origin/master)
Author: IZABELA SZCZEPANIK <izabela.szczepanik@ing.com>
Date:   Tue Nov 27 14:28:32 2018 +0100

    Iniszial komit mesydż

Na koniec podejrzyj historię. Użyj polecenia git log, zobacz także log w dowolnym GUI.
Jeśli nie zainstalowałeś jeszcze żadnego GUI, skorzystaj ze zintegrowanego z gitem gitk. Wpisz po prostu w konsoli gitk.
